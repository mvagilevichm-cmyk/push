# 🚀 Cargo Sender — Інструкція деплою на Railway

## Крок 1 — Завантаж файли на GitHub

1. Зайди на https://github.com і створи новий репозиторій (New repository)
2. Назви його `cargo-sender`, зроби Private
3. Завантаж всі файли з цієї папки в репозиторій

## Крок 2 — Підключи Railway

1. Зайди на https://railway.app
2. New Project → Deploy from GitHub repo
3. Вибери репозиторій `cargo-sender`
4. Railway автоматично визначить Python проект

## Крок 3 — Додай змінні середовища

В Railway → твій проект → Variables → Add Variable:

```
TG_API_ID      = 22703500
TG_API_HASH    = (твій хеш з my.telegram.org)
TG_PHONE       = +380633885088
SECRET_KEY     = будь-який_секретний_рядок_2026
```

⚠️ TG_API_HASH не публікуй — тримай в секреті!

## Крок 4 — Деплой

Railway автоматично задеплоїть після додавання Variables.
Чекай 2-3 хвилини — побачиш зелений статус.

## Крок 5 — Отримай URL

Railway → твій сервіс → Settings → Domains → Generate Domain
Отримаєш URL типу: `https://cargo-sender-production.up.railway.app`

## Крок 6 — Авторизація Telegram

1. Відкрий свій URL в браузері
2. Перейди в Налаштування (остання вкладка)
3. Введи номер телефону → натисни "Отримати код"
4. Введи код з Telegram → "Підтвердити"
5. Статус стане ✅ Підключено

## Готово! 🎉

Тепер додаток працює 24/7 навіть коли ПК вимкнений.
Відкривай з телефону по своєму Railway URL.

## Структура файлів

```
cargo-sender/
├── app.py              ← головний сервер
├── requirements.txt    ← залежності Python
├── Procfile            ← команда запуску
├── railway.json        ← конфіг Railway
├── .gitignore          
└── static/
    └── index.html      ← веб-інтерфейс
```

## Як працює розсилка

- Сервер перевіряє чати кожні 30 секунд
- Якщо чат готовий (пройшов ліміт) — відправляє повідомлення
- Тип 1 (Заявки → Перевізники) і Тип 2 (Реклама → Клієнти) — одночасно
- Авто-режим: сам запускається о 08:00 і зупиняється о 19:00
